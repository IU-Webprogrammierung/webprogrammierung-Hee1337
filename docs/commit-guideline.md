# Commit-Guidelines

Für dieses Projekt werden sprechende Commit-Messages verwendet, damit der Entwicklungsprozess nachvollziehbar bleibt.

## Verwendete Commit-Kategorien

- `chore:` technische Aufgaben, Projektstruktur, Setup
- `feat:` neue Funktionen oder neue Seitenbereiche
- `content:` Inhalte und Texte ergänzt oder überarbeitet
- `style:` CSS, Layout und visuelle Gestaltung
- `a11y:` Maßnahmen zur Barrierefreiheit
- `docs:` Dokumentation, Konzept, README
- `fix:` Fehlerbehebungen
- `refactor:` Code verbessert, ohne die sichtbare Funktion zu verändern

## Beispiele

```bash
chore: create initial project structure
feat: add semantic html structure
content: add introduction section
a11y: add skip link and main navigation
docs: add commit guidelines
style: add mobile first base styles
fix: correct navigation anchor link
refactor: simplify section structure
